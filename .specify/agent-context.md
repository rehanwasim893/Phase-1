# Agent Context: Todo In-Memory Python Console Application

## Technology Stack
- Python 3.13+
- Command-line interface (CLI) development
- In-memory data structures (lists, dictionaries)
- Modular application architecture
- PEP8 compliant code standards

## Architecture Patterns
- Layered architecture (CLI → Service → Model → Storage)
- Separation of concerns
- Single responsibility principle
- Clean code practices

## Key Components
- Todo data model with validation
- Service layer with business logic
- Storage layer with in-memory operations
- CLI layer with user interaction
- Main application loop

## Development Practices
- Input validation at appropriate layers
- Error handling with user-friendly messages
- Modular code organization
- Testable components
- Extensible design for future phases

## Constraints
- Python 3.13+ only
- UV package manager
- No external frameworks
- In-memory storage only
- CLI interface only
- PEP8 compliance required
